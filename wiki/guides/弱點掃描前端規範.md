# 前端弱點掃描規範 SOP

## 前置條件
- WASP ZAP 安裝（公司主要掃描工具）
- CSP Evaluator（需設定 HTTP Header，meta 無法偵測）

## 步驟

### 1. 設定 CSP meta
```html
<meta http-equiv="Content-Security-Policy" content="
  default-src 'self' 'unsafe-inline';
  object-src 'none';
  frame-src 'self' https://www.youtube.com;
  img-src 'self' data:;
  style-src 'self' https://fonts.googleapis.com;
  font-src 'self' data: https://fonts.googleapis.com https://fonts.gstatic.com;
  form-action 'self';
">
```

### 2. 加入 nonce（前端模板已設定 `nonce="crypto"`，後端套版時替換為實際值）
```html
<style nonce="{後端生成的隨機碼}"> ... </style>
<script nonce="{後端生成的隨機碼}"> ... </script>
```

### 3. 逐項檢查清單
- [ ] 無 CDN 引入（若必須用 CDN，在 CSP 加白名單）
- [ ] 無 inline style（例外：後台編輯器，需與客戶說明）
- [ ] 無 HTML `onClick` 屬性
- [ ] 無 `href="javascript:void(0)"`（改用 `href="#"` 或 `type="button"`）
- [ ] `target="_blank"` 已加 `rel="noopener noreferrer"`
- [ ] `innerHTML` 已改為 `createElement`
- [ ] 套件版本非過舊、非 beta

## 注意事項
- 弱掃通常不掃需登入的內頁，以頁面連結路徑掃描
- 一般只需處理中、高風險警示
