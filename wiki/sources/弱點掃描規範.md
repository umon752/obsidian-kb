# 弱點掃描規範

## 核心要點
- 工具：WASP ZAP（公司主要使用）、CSP Evaluator

### 前端必須遵守規範
- 在 `<meta>` 設定 CSP，避免使用 `unsafe-inline` 與 `unsafe-eval`
- `<script>` 與 `<style>` 加上 `nonce="後端生成隨機碼"`
- 禁止使用 CDN（防止第三方篡改）
- 禁止 inline style（例外：後台編輯器產生的樣式需與客戶說明）
- 禁止 HTML 標籤使用 `onClick`
- 禁止 `href="javascript:void(0)"`
- `target="_blank"` 連結加上 `rel="noopener noreferrer"`
- 盡量改用 `createElement` 取代 `innerHTML`（防 XSS）
- 套件需注意版本不能太舊、不使用 beta 版

### 常用 CSP 範本
```
<meta http-equiv="Content-Security-Policy" content="default-src 'self' 'unsafe-inline'; object-src 'none'; frame-src 'self' https://www.youtube.com; img-src 'self' data:; style-src 'self' https://fonts.googleapis.com; font-src 'self' data: https://fonts.googleapis.com https://fonts.gstatic.com; form-action 'self';">
```
