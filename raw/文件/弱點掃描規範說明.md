# 弱點掃描規範說明

近期因為弱點掃描有更新改版的關係，規範比以前嚴謹，所以需要前端這邊切版的時候把弱掃資安的部份考慮進來，即便案子沒有弱掃的需求，也一併執行。

## 檢測工具

> 一般弱點掃描不會掃到會員登入等內頁，掃描方式是以頁面跟頁面之間的連結去跑。
> 
- [WASP ZAP](https://www.zaproxy.org/) (需下載軟體並安裝) [入門](https://www.zaproxy.org/getting-started/)
公司通常是使用這套軟體掃描產出報告，一般只需處理中、高風險的警示即可。
- [CSP Evaluator](https://csp-evaluator.withgoogle.com/) (需把 CSP 放到 HTTP Header，meta 會捉不到)

## 前端執行規範

- 在 meta 內設定 CSP
    - CSP 是瀏覽器提供網站設定白名單的機制，網站可以告知瀏覽器，該網頁有哪些位置可以連、哪些位置不能連
    - CSP 在 meta（前端）和 HTTP Header（後端）皆可設定，CSP 指令可以限制發出 Request 獲取資源的類型及位置，設定之後如果有出現不合規範、有資安疑慮的寫法，console 就會報錯
    - 設定 CSP 不可太寬鬆，如果設定太寬鬆，console 雖然都沒事，但是弱掃的時候還是有可能會被掃出來，避免使用 `unsafe-inline` 跟 `unsafe-eval`
    - [參考文章](https://ithelp.ithome.com.tw/articles/10196896)
    
    ```
    <meta http-equiv="Content-Security-Policy" content="{CSP 指令} {位置}; {CSP 指令} {位置} {..位置..} {位置};">
    
    ```
    
- `<script>` 和 `<style>` 皆需加上 `nonce="後端生成的隨機碼"`
    - 已設定在前端模板內 `nonce="crypto"` ，後端套版時會把 `crypto` 改成實際隨機碼的變數
    - [MDN](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/nonce)
        
        ```
        <style nonce="{後端生成的隨機碼}">
         .el {
            color: red;
          }
        </style>
        
        <script nonce="{後端生成的隨機碼}">
          const el = document.querySelector('.el')
        </script>
        
        ```
        
- 禁止使用 CDN
    - 防止 CDN 來源的檔案被 CDN 業者篡改
    - 如果遇到只能使用 CDN 引入的套件或 api，設定範例如下(以 google font 為例)： { CSP 指令 } { 只允許同源的來源 } { 指定來源: } { URL } ; `font-src 'self' data: [<https://fonts.googleapis.com>](https://fonts.googleapis.com/) [<https://fonts.gstatic.com>](https://fonts.gstatic.com/)`
- 使用套件需要注意版本不能太舊、不要使用到 beta 版 有些套件會相依別的套件，相依的套件如果還是比較舊的版本的話，這個套件可能就會無法使用了
- 禁止使用 inline style
    - 文章編輯器，使用者在後台設定的樣式，在前台會以 inline style 的方式生成，這題有點無解，只能請專案去跟客戶說明，若客戶可以接受，弱掃的這條警示就可以忽略
    - 如果是透過 js 動態去修改元素樣式，盡量以 class 的方式處理 (但其實這個弱掃應該是沒問題的 (?)
    - 有些套件生成時會自帶 inline style，解決方法：可以將生成的樣式拉出來放到自己的 sass 內，或獨立出一個 css 檔引用
- 禁止在 HTML 標籤上使用 onClick
- 禁止使用 `href="javascript:void(0)"`
    - HTML 標籤上不可出現 javascript 字眼，弱掃會以為這個可能是有執行作用的東西
- a link `target="_blank"` 外連時加 `rel="noopener noreferrer"`
    - 防止點網址開新視窗時原網頁被切換篡改
    - [參考文章](https://pjchender.blogspot.com/2020/05/relnoreferrer-targetblank.html)
        
        ```
        <a rel="noopener noreferrer" target="_blank" href="<http://example.com/>">連結</a>
        
        ```
        
- 盡量禁止 innerHTML，改用 createElement
    - 避免外來的 HTML 標籤輸入值造成 XSS 攻擊
        
        ```
        原範例：
        el.innerHTML = "<img src="text.jpg>";
        更改後範例：
        const img = document.createElement('img');
        img.src = 'xyz.jpg';
        
        ```
        

> 通常會設定的 CSP 範例 (已放在前端模板內)
> 

```
<meta http-equiv="Content-Security-Policy" content="default-src 'self' 'unsafe-inline'; object-src 'none'; frame-src 'self' https://www.youtube.com; img-src 'self' data:; style-src 'self' https://fonts.googleapis.com; font-src 'self' data: https://fonts.googleapis.com https://fonts.gstatic.com; form-action 'self';">

```
