## 規範

- 需過無障礙 WCAG 2.2 AA（使用 AQA 檢測軟體檢查）
- 自訂最外層 class / id（名稱不衝突於華碩本身網站即可），所有 class / html tag 選取都需指定在這個 class 下，避免被華碩本身引入的 css 覆蓋（尤其是 html tag 的部份，h2~h6、a tag 需要特別注意一下）
- 先確認放上官網的頁面本身有沒有使用 `<h1>`，如果沒有要記得設定 (ex：官網已有 h1 所以從 h2 開始寫)
- Focus 鍵盤觸發視覺性
- 影片自動播放 5 秒後停止（超過5秒要有停止/播放按鈕）
- alt 描述字元建議盡量不可超過 125 個字元、精簡為主（ADA、SEO規範）
- 圖片使用 webp，圖片、影片使用 lazyload 降低初始流量
- **hover 狀態、fade in 效果、輪播切換效果等皆要注意色彩對比度**
使用偽元素設定的背景顏色 AQA 會忽略它，必須使用元素本身或父層元素本身設定的背景色為準
- 需要思考到有可能會新增、刪除、可編輯的地方（ex：提供他們 local 改語系），文字長短版面上的考量 (設計在設計時務必思考進來)
- 不可在檔案後方加上版本號 (這樣會更動到 HTML，華碩他們清快取會有問題，local 會無法即時更新)
分版本以資料夾來區分檔案 (ex: /v1、/v2)
只要更新 file 都會是統一 global 一起更新，所以如果是已經 publish 的產品要特別注意，不同版本一定要另外用資料夾區分更新檔案
**更新 file 檔案有時會遇到快取問題，請試著再次更新 檔案，試到 preview 正確讀到新檔案為止**，如果真的清不了再請華碩他們處理
- 影片需縮小到 5MB 以內
影片需加上 poster 預設圖、文字描述檔 .vtt
    
    .vtt 檔參照以下格式：
    
    (第 1 行) 開頭 `WEBVTT`，不用更動
    (第 3 行) 時間軸 `hh:mm:ss.mmm --> hh:mm:ss.mmm`，請查看該影片時長，修改箭頭後方的秒數
    (第 4 行) 字幕內容，貼上客戶提供的文字描述
    
    ```
    WEBVTT
    
    00:00:01.000 --> 00:00:04.000
    A person sitting at a table, with the camera looking over their shoulder and using an ROG Xbox Ally X.
    
    ```

## 打包原始碼

華碩會用本機直接開啟檔案預覽，非 live server 方式預覽，所以只要有使用 ES6 import 和 ajax 等等的方法，就會產生跨域 CORS 的問題，因此使用 ajax 載入 json 的部份需改寫成 js 檔，用 import 進來的方式處理，最後需用 babel 進行轉譯。
